Detailed Description

The ThreadPool class implements a simple pool of working threads. At construction, <n> threads are spawned and tries to get tasks to execute from a queue. Any access to the queue is synchonized through the use of a mutex. Jobs are added to the queue by calling addJob(task) where task is the result of std::bind(). The queue can be open - in which case the jobs are effectively added to the queues - or closed - in which case nothing can be push into the queue anymore. Stopping the pool is done through calling join() which will close the queue.
